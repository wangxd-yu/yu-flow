# 引擎架构与介绍

Yu Flow 是一款纯正的**企业级全栈嵌入式低代码中间件**。

它的设计初衷，是为了把繁重、重复的 CRUD 接口开发及简单的中后台 UI 交互全部配置化，让研发工程师将核心关注点回归到具有高商业价值的计算与核心系统流程。

## 为什么选择嵌入式底座？

业内绝大多数“低代码平台”都是重量级的全家桶。它们要求您的业务方去适应它，甚至需要全盘迁移。 
Yu Flow 提供的是**底座级 Starter**。
您只需在现有的 Spring Boot 项目中引入 `yu-flow-api` 依赖，所有的低代码能力立刻通过 `@EnableYuFlow` 赋能在您的项目中，天然与您当前系统的 Redis、数据库连接、安全鉴权体系保持融洽。

## 技术栈与底层驱动

本系统在底层构建依赖业界领先且完全开源的基石：

- **服务端内核 (`flow-api`)**
  - **核心引擎**：Spring Boot 2.x
  - **动态安全**：基于 `HandlerInterceptor` 和高速多级内存（如 5 秒 SPA 缓存拦截器）提供最极端的接口高并发承载力。
  - **动态语法树**：底层运用了 SpEL（Spring 表达式语言）处理所有复杂的动态变量和安全映射。
- **前端生成器 (`flow-ui`)**
  - **交互框架**：蚂蚁金服 [UmiJS/Max](https://umijs.org/) 搭配 React 18。
  - **低代码协议**：深度集成百度 [Amis (v6.13+)](https://aisuda.bce.baidu.com/amis/) 作为 UI JSON 解析的核心渲染器，并通过 `amis-editor` 提供所见即所得的极智设计体验。
  - **画布编排**：依赖 `@dnd-kit/core` 与 `@antv/x6` 实现严谨的数据流向图谱。

## 典型实施链路

一个完整的业务需求在 Yu Flow 中的实现流程通常是这样的：

1. **连接数据库**：在【数据源管理】中配置 JDBC URL 和安全证书。
2. **抽象元数据**：在【数据模型管理】中，引擎自动为您扫描出物理表（如 `t_student`），您为其配置“学生名称”、“年龄”以及它的校验规则（邮箱验证、非空等）。
3. **点亮接口**：在【API 配置管理】中点击生成。引擎将在内存态虚拟出真正的 `POST /flow-api/dynamic/student`。
4. **组装视图**：在【页面可视化管理】使用 Amis Editor，通过简单拖拽输入上述 API。一键发布，您的用户便看到了精美的业务系统！
