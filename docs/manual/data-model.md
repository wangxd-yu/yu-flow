# 数据模型管理

Yu Flow 的元数据驱动底座核心：**配置物理表向逻辑视图的映射**。

通过数据建模，业务逻辑可以脱离“SQL 增删改查”思维，转而以对象和属性（Field Schema）的维度进行定义和管理，并能在前端直接反映出所见即所得的表单或列表形态。

## 模型定义

在操作后台导航到 【数据模型】 (`/data-model/list`)，您可以看到企业域内所维护的业务模型（如 `学生配置`, `接口管理`, `用户配置`）。每个逻辑模型都有：
- **物理表名**：例如对应的数据库引擎底层的 `t_student` 或平台核心表 `flow_datasource`。
- **目标数据源**：选择此张表存放于上方建立的哪一个 JDBC Datasource（支持不同物理库跨域挂载）。

## 核心配置流

- **配置字段（元数据驱动层）**  
  点击配置动作，为您的物理表增加业务规则语义：不仅包含字段英文名、显示备注，还能设置**校验器**（例如是否必填，是否限制为 Phone / Email / Custom RegEx 格式验证）。
  > 后端的模型会自动转译到 Amis 侧的 `validations` 和 `rules`，实现端到端的联动校验。

- **一键生成页面 / 智能生成 CRUD 视图**  
  当建立好模型映射后，您可以点选“生成页面”，极简模式下我们为您自动化打通 `Table` 的搜索组件条件和渲染展现形式。
