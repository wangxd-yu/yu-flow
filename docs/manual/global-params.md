# 全局参数与宏注入

为了解决动态 API 在运行期所遇到的一些复杂环境常量诉求（例如生成订单号所需的时间戳、获取分布式唯一 ID 等），引擎提供了**全局参数中心（宏指令）** `/sys-macro/manage`。

## 什么是全局参数？

全局参数是引擎在执行期挂载在上下文（Context）中的变量和内置方法引用。其底层表达式解释器依赖 `SpEL (Spring Expression Language)` 机制。
在【非高级管理员请勿修改】的安全提示下，您在这里定义了整个引擎运行时的“词典”。

## 系统预置宏一览

我们已经为您在启动前预置了常用的业务逻辑变量（可以在动态 API 输入参数及数据模型默认值中随意采用）：

- **分布式序列号发生器**
  - **`UUID`** (32位无连字符)：生成用于主键的 32 字符标识。
  - **`SNOWFLAKE`** (雪花算法)：提取自高性能发号器算法，用于高并发下绝对保证时序性和唯一性的 ID。

- **时间上下文**
  - **`DATE_TIME_FULL`**：精准提取完整日期时间到毫秒（格式 `yyyy-MM-dd HH:mm:ss.SSS`）。
  - **`DATE_TIME`**：截取标准日期时间串 (`yyyy-MM-dd HH:mm:ss`)。
  - **`TIME`** / **`DATE`**：仅供提取时/分/秒，或者当前自然日的字符串。

- **多态与环境依赖（方法级提取）**
  - **`GET_ENV`**：动态从部署底座获取 Spring Environment 变量。在运行期可自动适配 Dev / Prod 环境。
  - **`DATE_FORMAT`**：提供标准的格式化入参计算方法。

这些参数作为动态语法的顶层字典，将在您所有生成的 API 请求中发挥“魔法渲染”作用！
