# 🎨 编排画布与核心概念总览

**欢迎来到 Yu Flow 的指挥中心。** 在这里，你所有的业务逻辑都将像搭积木一样完成拼接——从左侧拖出一个节点，往中间画布一放，再拉一条线连到下游，一条完整的业务接口就搭建完成了。不需要写代码，不需要等发版。

本指南将帮助你快速理解编排画布的核心概念。读完之后，你将能够自信地完成任何节点的配置和连线。

---

## 1. 认识你的工作台

打开任意一条 Flow 进入编辑模式后，你会看到一个由三大区域组成的经典编排界面：

### 📦 左侧：组件库 (Palette)

这里陈列着你可以使用的所有**节点类型**，按功能分类整齐排列：

| 分类 | 包含的节点 | 一句话说明 |
| --- | --- | --- |
| **基础节点** | Request、Response | 流程的入口和出口 |
| **数据节点** | Database、SystemVar | 查数据库、取系统变量 |
| **逻辑节点** | If、Evaluate、Switch | 条件判断、表达式计算 |
| **调用节点** | HttpRequest、SystemMethod | 调外部接口、调系统工具方法 |
| **控制节点** | For、Collect | 循环、并行分支汇聚 |

**操作方式：** 从组件库中**拖拽**节点到中间画布即可。松手后节点就"落"在画布上了。

### 🗺️ 中间：画布区 (Canvas)

这是你的主战场——一个**无限滚动的白板**。

- **拖拽节点** → 调整位置
- **从端口拉线** → 建立节点之间的执行顺序
- **鼠标滚轮** (Ctrl/Cmd + 滚轮) → 缩放画布
- **空白处拖拽** → 平移画布
- **Delete / Backspace** → 删除选中的节点或连线
- **Ctrl+Z / Cmd+Z** → 撤销操作

::: tip 💡 小技巧：快捷添加
从一个节点的输出端口**拉线到空白处**松手，系统会弹出一个快捷组件选择菜单，让你直接选择要创建的下游节点——省去了"先拖节点再连线"的步骤。
:::

### ⚙️ 右侧：属性面板 (Properties)

点击画布上的任意节点，右侧会自动弹出该节点的**详细配置表单**。不同类型的节点有不同的配置项：

- Request 节点 → 配置 HTTP 方法、参数校验规则
- Database 节点 → 编写 SQL、选择数据源
- If 节点 → 编写条件表达式、选择语言
- Response 节点 → 设置状态码、响应头、响应体

> 点击画布空白处可以关闭属性面板。

## 2. 节点的公共属性

虽然每种节点的功能各不相同，但它们都共享以下基础属性：

### 🆔 节点 ID (Node ID)

::: warning ⚠️ 这是最重要的属性，没有之一！
**节点 ID 直接决定了下游节点如何找到并引用本节点产生的数据。**

例如，你把一个 Database 节点的 ID 设为 `query_user`，那么下游任何节点都可以通过 `$.query_user.out` 来获取它的查询结果。

如果你随意修改了某个节点的 ID，所有引用它旧 ID 的下游节点都会**取不到数据**！
:::

- **系统自动生成规则：** `{节点类型}_{时间戳}_{序号}`，例如 `database_1772498001234_1`
- **建议手动命名：** 给核心节点取一个语义化的 ID（如 `query_user`、`check_vip`），会让你的流程更易读、更容易维护

### 🏷️ 节点名称 (Label)

显示在画布卡片上的标题文字，**仅供人类阅读**，不影响任何运行逻辑。双击节点标题即可修改。

建议写成有业务含义的短语，例如：
- ✅ `查询VIP用户信息`
- ✅ `计算订单总价`
- ❌ `database_1` （没有任何辨识度）

### 📝 描述 (Description)

作为节点的备注信息，适合记录"这个节点为什么这样配置"的上下文说明。在团队协作中非常有用。

## 3. 连线与分支走向

节点之间通过**连线 (Edge)** 连接。连线从源节点的**输出端口**拉出，到目标节点的**输入端口**松手。

### 理解端口 (Port)

每个节点卡片上的**小圆点**就是端口。端口分两种：

| 位置 | 类型 | 作用 |
| --- | --- | --- |
| 节点**左侧** | 输入端口 (in) | 接收上游传来的控制流或数据 |
| 节点**右侧** | 输出端口 (out / true / false) | 将执行结果传递给下游节点 |

### 常见的输出端口类型

绝大多数节点只有一个 `out` 输出端口，但部分"分流型"节点有特殊的多端口：

| 端口名 | 颜色/样式 | 含义 |
| --- | --- | --- |
| **out** | 🟢 实心绿 | 标准成功输出（绝大多数节点） |
| **true** | 🟢 实心绿 | If 节点条件**成立**时走此路 |
| **false** | ⚪ 空心圆 | If 节点条件**不成立**时走此路 |
| **item** | 🟡 实心黄 | For 循环中每一项的子流程入口 |
| **done** | 🟢 实心绿 | For 循环结束后的流程出口 |

### 连线代表什么？

```
[ Database 查用户 ] ——out——→ [ If 判断VIP ] ——true——→ [ Database 查VIP折扣 ]
                                            ——false——→ [ Database 查普通价 ]
```

- 从 `out` 拉出的线 → **"执行完当前节点后，去执行下一个节点"**
- 从 `true` 拉出的线 → **"条件成立时走这条路"**
- 从 `false` 拉出的线 → **"条件不成立时走这条路"**

::: tip 💡 分支可以不全连
If 节点有两个输出端口 (true / false)，但你**不需要两条路都连**。如果只关心"条件成立"的情况，只连 true 端口即可，false 不连线时，这条分支会直接结束。
:::

## 4. 数据怎么流转？

这是本指南中**最重要的概念**。理解了它，你就掌握了 Yu Flow 的精髓。

### 🚂 火车比喻

你可以把整个 Flow 想象成一列行驶中的火车：

> **Context（上下文）** 就是火车的 **货厢**。

- 火车沿着铁轨（连线）依次经过每一站（节点）
- **每个节点执行完毕后**，都会把自己的结果打包成一个**包裹**，扔进货厢里
- 包裹上贴着自己的名字——就是**节点 ID**
- **下游的任何节点**，只要知道包裹的名字和里面的格式，就可以从货厢里取出需要的数据

### 📦 提取码语法

下游节点从上下文中取数据时，使用统一的 **JSONPath 提取语法**：

```
$.节点ID.out
```

**完整示例：**

假设流程中有一个 ID 为 `query_user` 的 Database 节点，它查询了一条用户记录：

```json
{ "id": 1001, "name": "张三", "vip_level": 3 }
```

那么下游节点可以这样引用：

| 提取码 | 得到的值 |
| --- | --- |
| `$.query_user.out` | 整个用户对象 `{ id: 1001, name: "张三", ... }` |
| `$.query_user.out.name` | `"张三"` |
| `$.query_user.out.vip_level` | `3` |

::: info 📌 小结
1. 每个节点执行完，结果自动存入 Context
2. Key 就是节点的 **ID**
3. 下游通过 `$.节点ID.out` 扫码取件
4. 所以**给核心节点取一个好认的 ID** 真的很重要！
:::

### 连线自动绑定

在画布上拉线连接两个节点时，系统会**自动帮你填写 extractPath**。

例如：当你从 `query_user` 的 `out` 端口拉线到 If 节点的某个变量输入端口时，系统会自动将该变量的提取路径设为 `$.query_user.out`——你不需要手动输入任何路径。

## 6. 动态表达式与系统能力

在 Yu Flow 中，很多配置项不仅能填写固定值，还支持**动态表达式**。

### `${}` 变量引用

在 Database 的 SQL、Response 的 Body 等文本输入框中，使用 `${变量名}` 语法来引用上下文中的动态数据：

```sql
SELECT * FROM users WHERE id = ${userId} AND status = ${status}
```

引擎会在运行时将 `${userId}` 替换为上游节点传入的实际值。

### 表达式语言

If 和 Evaluate 节点支持在表达式中编写更复杂的逻辑，可选择三种语言引擎：

| 语言 | 语法示例 | 适用场景 |
| --- | --- | --- |
| **JavaScript** | `age >= 18 && status === 'active'` | 最通用，语法最熟悉 |
| **Aviator** | `age >= 18 && status == 'active'` | 高性能规则计算 |
| **SpEL** | `#age >= 18 && #status == 'active'` | 需调用 Spring Bean 时 |

### 系统内置能力

Yu Flow 预置了丰富的系统变量和工具方法，让你无需写代码就能完成常见操作：

::: info 🔗 延伸阅读
- 👉 **[系统变量 (SystemVar) 节点指南](/api/nodes/system-var)** — 获取当前登录用户、时间戳、雪花ID 等系统级数据
- 👉 **[系统方法 (SystemMethod) 节点指南](/api/nodes/system-method)** — 调用日期格式化、加密解密、字符串处理等工具方法
:::
